Основы работы с Git

Что такое система контроля версий и зачем она нужна?

Ответ: Система контроля версий - это специальное ПО, которое позволяет сохранять проект на разных этапах, и при нужде, возвращатся к предыдущим версиям. Также оно позволяет организовать работу в команде, благодаря системе веток, один красноглазик делают один сервис, другой другой, и потом вместе сливают ветки, и получают проект, в котором имплементированны оба сервиса.

Что делает команда git clone?

Ответ: Клонирует репозиторий через ссылку.

Что делает команда git init?

Ответ: Инициализирует новый репозиторий в папке проекта, в которой открыт терминал.

Что такое коммит и для чего он используется?

Ответ: Коммит - это фиксация изменений в репозитории. Он используется для сохранения текущего состояния проекта.

Как добавить изменения в индекс перед коммитом?

Ответ: Через команду git add.

Для чего используется команда git checkout?

Ответ: Команда git checkout используется для переключения между ветками, возращения к предыдущим коммитам, и для отмены изменений.

Что означает понятие "Ветка" в Git?

Ответ: "Ветка", или же по англо-саксонски "branch" в Git - это изолированная ветка проекта. Таких веток может быть множество. Они позволяют независимо от друг друга, разрабытать разные функции в проекте, и в конце либо объединить ветки, либо же удалить одну из них.

Как создать новую ветку в Git?

Ответ: Через команду git branch.

Что делает команда git push?

Ответ: Позволяет залить коммиты из локальной машинки, на удалённый репозиторий.

Что делает команда git pull?

Ответ: Загружает изменения с удалённого репозитория, и фиксирует их на локальном проекте.

Для чего используется README файл в репозитории?

Ответ: Это файл на языке разметки, в котором указывается необходимая информация.

Для чего используется файл .gitignore?

Ответ: Для исключения файлов и папок, которые не должны фиксироватся.

Что такое SSH ключи и зачем они нужны?

Ответ: Это связка двух криптографических ключей, публичного и приватного. Публичный ключ позволяет зашифровать соединение с сервером, чтобы никто из в открытую не мог ничего прочесть. Он хранится и на клиенте, и на сервере. Приватный же ключ хранится только на клиенте, и позволяет расшифровать данные, а также используется для аутетификации. Они нужны для обеспечения безопасного соединения между сервером и клиентом.

Что такое GitHub Pages и как его использовать?

Ответ: Это бесплатный хостинг, для сайтов без бекэнда, то бишь только статика и вёрстка. Для его использования нужно для начала создать репозиторий, в котором и будут распологаться нужные файлы, и после этого на страничке репозитория, в разделе settings будет пунктик pages в нём надо выбрать ветку, сохранить всё, и через время, сайт развернётся и появится ссылка, по которой он будет доступен.
